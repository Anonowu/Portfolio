Overview

About the project

DataFlow's product suite had grown to 6 product teams, each building UI independently. The result: 12 button styles, 8 shades of blue, and 0 shared components. I was brought in to design and ship a company-wide design system from scratch in 6 months.


Orbit v1 launched with 240 components across 8 categories, with full dark mode and multi-brand token support. It's still actively maintained and has become the foundation for all new product development.

The Problem

The chaos before Orbit

Without a shared system, each team was rebuilding the same components independently. A "secondary button" looked different in every product. Designers were spending 60% of their time on UI maintenance rather than UX work.


Engineers estimated 30% of their sprint time was spent on UI inconsistency tickets. It was a tax on the entire organisation.

  • 12 different button styles across 6 products
  • 8 different shades of "brand blue"
  • 60% of design time on UI maintenance
  • 30% of engineering sprint on inconsistency bugs
  • No shared Figma file or source of truth

Architecture

How Orbit is structured

01
🎨
Token Layer
Base → Semantic → Component tokens. Multi-brand support via theme overrides. Light and dark mode built in.
02
🧱
Primitive Components
Atoms: Button, Input, Badge, Icon, Avatar. All token-driven. 100% Figma-to-code parity via Storybook.
03
🔧
Composite Components
Molecules: Card, Modal, Form, Table, Navigation. Built from primitives. Variants cover all edge cases.
04
📚
Documentation
Every component has usage guidelines, accessibility notes, code snippets, and a do/don't guide.

Reflections

What I learned

📣
Adoption requires advocacy
The best design system in the world fails if teams don't use it. I embedded "system advocates" in each team and held monthly office hours. Adoption is a product problem, not just a design problem.
🔑
Token naming is a decision
Semantic token names like "color.primary" beat literal names like "color.green" every time. When the brand colour changes, semantic names mean zero component updates required.
🚢
Ship imperfect, iterate fast
We shipped v1 at 80% complete. The 20% we were still building became v1.1 and v1.2. Waiting for perfection delays value. Teams needed a foundation to build on, not a perfect system.
